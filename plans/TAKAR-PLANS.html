Latar Belakang

Polymarket mencatatkan volume trading $8 miliar di 2024. Belum ada platform sejenis untuk pasar Indonesia — padahal kebutuhan akan informasi terukur dan prediksi probabilistik sangat tinggi.

🎯 Visi

Membangun platform prediction market pertama Indonesia — TAKAR — di mana siapapun bisa memperdagangkan opini mereka tentang hasil kejadian nyata secara terukur, transparan, dan bertanggung jawab.

❓ Masalah yang Dipecahkan

Opini publik di Indonesia tersebar tanpa ukuran probabilitas yang jelas. Tidak ada mekanisme price discovery untuk prediksi kejadian — politik, ekonomi, olahraga, teknologi. TAKAR mengisi gap ini.

💡 Inspirasi & Diferensiasi

Terinspirasi Polymarket, namun dibangun khusus untuk konteks Indonesia: Rupiah, konten lokal (Pilkada, IHSG, Liga 1, startup unicorn), antarmuka Bahasa Indonesia, dan jalur regulasi sesuai hukum RI.

Database Schema

PostgreSQL sebagai primary database dengan event sourcing via CosmosDB untuk audit trail lengkap setiap transaksi.

SQL — PostgreSQL 
-- ─────────────────────────────────────────
-- Core Tables · TAKAR Platform
-- ─────────────────────────────────────────

CREATE TABLE users (
  id          UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  username    VARCHAR(50)  UNIQUE NOT NULL,
  email       VARCHAR(255) UNIQUE NOT NULL,
  balance_idr BIGINT       DEFAULT 0,        -- dalam sen (IDR × 100)
  kyc_level   SMALLINT     DEFAULT 0,        -- 0=anon, 1=email, 2=KTP
  created_at  TIMESTAMPTZ  DEFAULT now()
);

CREATE TABLE markets (
  id            UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  title         TEXT        NOT NULL,
  description   TEXT,
  category      VARCHAR(50),                    -- politik, ekonomi, olahraga
  resolution_at TIMESTAMPTZ NOT NULL,
  resolved_at   TIMESTAMPTZ,
  outcome       BOOLEAN,                        -- null = belum resolved
  liquidity_b   BIGINT      DEFAULT 1000000,  -- LMSR parameter b (sen)
  status        VARCHAR(20) DEFAULT 'active',
  created_by    UUID        REFERENCES users(id),
  created_at    TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE positions (
  id         UUID    PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID    REFERENCES users(id),
  market_id  UUID    REFERENCES markets(id),
  outcome    BOOLEAN NOT NULL,              -- YES or NO
  shares     BIGINT  NOT NULL,              -- jumlah shares
  avg_cost   BIGINT  NOT NULL,              -- harga rata-rata beli (sen/share)
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE trades (
  id         UUID       PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID       REFERENCES users(id),
  market_id  UUID       REFERENCES markets(id),
  side       VARCHAR(4) NOT NULL,           -- 'BUY' or 'SELL'
  outcome    BOOLEAN    NOT NULL,
  shares     BIGINT     NOT NULL,
  price      BIGINT     NOT NULL,           -- total cost in sen
  fee        BIGINT     NOT NULL,           -- platform fee in sen
  created_at TIMESTAMPTZ DEFAULT now()
);
06

Mekanisme Pasar (LMSR)

TAKAR menggunakan Logarithmic Market Scoring Rule (LMSR) — mekanisme AMM yang terbukti efisien untuk prediction market tanpa memerlukan counterparty langsung.

Cost Function (LMSR)
C(q_yes, q_no) = b × ln( e^(q_yes/b) + e^(q_no/b) )
Harga Probabilitas YES
P(YES) = e^(q_yes/b) / ( e^(q_yes/b) + e^(q_no/b) )
Biaya Pembelian Δq Shares
cost = C(q_yes + Δq, q_no) − C(q_yes, q_no)

✅ Keunggulan LMSR

Selalu ada likuiditas (tidak perlu matching buyer-seller), harga mencerminkan probabilitas pasar secara natural, maksimum loss platform terbatas pada nilai parameter b.

⚙️ Parameter b

Semakin besar b, semakin likuid pasar tapi semakin besar modal yang dikunci. MVP: b = Rp 1.000.000 per pasar. Scaling berdasarkan interest & volume historis tiap kategori.

🔮 Resolusi

Outcome ditentukan oleh panel oracle internal + data sumber terpercaya (BPS, KPU, PSSI, Reuters). Pemenang mendapat 1 share = Rp 1.000. Platform mengambil fee 2% dari setiap trade.

08

Regulasi & Kerangka Hukum

Tantangan terbesar. Strategi utama: framing sebagai "skill-based forecasting" bukan perjudian, dimulai dengan virtual currency.

⚠️ Risiko Regulasi Utama

UU No.7 Tahun 1974 tentang Penertiban Perjudian dan KUHP Pasal 303 melarang aktivitas yang dapat dikategorikan judi. Platform perlu struktur legal yang secara jelas memisahkan diri dari definisi tersebut.

🛡️ Strategi Mitigasi Legal

1) Mulai dengan virtual currency (tidak ada uang nyata di fase awal). 2) Framing sebagai platform edukasi & riset opini. 3) Daftarkan ke OJK Regulatory Sandbox. 4) Konsultasi kuasa hukum spesialis fintech.

📋 Jalur Regulasi Bertahap

Phase 1: Virtual points only (tidak perlu izin khusus). Phase 2: OJK Sandbox sebagai inovasi keuangan digital. Phase 3: Guidance resmi dari OJK dan Kemenkominfo.

🏦 Aspek Keuangan

Jika melibatkan uang nyata: perlu izin sebagai Penyelenggara Sistem Pembayaran atau kerjasama dengan lisensi eksisting. Model escrow via bank partner dapat menjadi solusi jangka menengah.

Roadmap

4 fase dari riset hingga scaling nasional. Setiap fase memiliki milestone jelas dan go/no-go decision point.

Phase 0 — Q2 2026 · Sekarang
Research & Validation
  • Riset regulasi mendalam dengan kuasa hukum
  • User interview 50+ target pengguna
  • Prototype desain (Figma) + user testing
  • Setup legal entity & konsultasi OJK
  • Finalisasi arsitektur & tech stack
Phase 1 — Q3 2026
MVP Build (Virtual Points)
  • Core platform: auth, markets, LMSR trade engine
  • 10–20 markets percontohan (IHSG, isu nasional)
  • Closed beta: 500 undangan komunitas
  • Tidak ada uang nyata — virtual points saja
  • Feedback loop & iterasi cepat
Phase 2 — Q4 2026 – Q1 2027
Public Beta + Payment
  • Integrasi Midtrans / Xendit (top-up & withdrawal)
  • KYC level 1 (email verification)
  • Open registration — target 10.000 users
  • OJK Sandbox application
  • Launch B2B data API (media partner pertama)
Phase 3 — Q2–Q3 2027
Growth & Monetization
  • Target 100.000 registered users
  • Mobile app (React Native)
  • Sponsored markets dengan brand & media besar
  • Premium subscription launch
  • Fundraising seed round
